# v0.10.6 — license switched to Apache 2.0

Patha is now licensed under the Apache License, Version 2.0 (was MIT in v0.10.5).

## Why Apache 2.0

MIT is permissive, well-understood, and friction-free — and so is Apache 2.0. The substantive differences favour Apache 2.0 for a project with genuinely novel architecture:

| | MIT | Apache 2.0 |
|---|---|---|
| Permissive (use, modify, redistribute, commercial) | ✅ | ✅ |
| Explicit patent grant from contributors | ❌ silent | ✅ explicit |
| Patent retaliation (anyone who patent-trolls Patha loses their license) | ❌ none | ✅ §3 |
| Structured attribution (`NOTICE` file) | ❌ | ✅ |
| Modern default for AI/ML projects (TensorFlow, Kubernetes, Mem0, Letta) | sometimes | usually |
| Enterprise legal-team friction | low | even lower (patent terms are explicit) |

Patha implements genuinely novel architecture — synthesis-intent routing, gaṇita as a deterministic recall strategy with zero LLM tokens, non-commutative belief evolution, the Vedic 7-view encoding, songline graph traversal. Explicit patent terms matter more for that than for a generic utility library.

The other realistic options were considered and rejected:
- **AGPL 3** — too restrictive. Many enterprises ban AGPL outright; would erase the "drop into Claude Desktop / Cursor / Zed / Goose" pitch for a real fraction of users. Patha's value lives in being installable everywhere.
- **Source-available (BSL, Elastic License, SSPL)** — post-success defensive moves applied prematurely just suppress growth.
- **Dual-licensing (AGPL + commercial)** — only worth the legal overhead when there's measurable commercial demand to capture. Not the situation today.

## What changed

- **`LICENSE`** — added with the canonical Apache 2.0 text. (The v0.10.5 README badge linked to `LICENSE` but no such file actually existed in the repo. Fixed alongside the switch.)
- **`NOTICE`** — added per Apache 2.0 convention. Carries the project copyright (`Copyright 2026 Stefi P. Krishnan`), attribution, and a structured pair-programming credit for Claude Code.
- **`pyproject.toml`** — `license = "Apache-2.0"`; classifier `License :: OSI Approved :: Apache Software License`.
- **`README.md`** — license badge updated, "## License" section now points at `LICENSE` + `NOTICE`.

No code or test changes vs v0.10.5. License is wheel metadata, so a version bump is the only way to propagate the change to PyPI.

## Notes for downstream consumers

Apache 2.0 is permissive — use, modify, redistribute, and commercial use are all allowed. The only practical obligation is preserving the `NOTICE` text and the Apache 2.0 copyright header in significant redistributions — both standard for any Apache 2.0 dependency you already use.

**v0.10.5 has been yanked from PyPI.** It carried an MIT license but is no longer available for new installs; pip will skip it during normal resolution. Use v0.10.6 (Apache 2.0).
