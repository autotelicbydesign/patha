## Architectural distinction (unchanged from v0.10.2)

`Memory.recall()` routes by question intent:

- **Retrieval** (*pratyakß╣Ża*) ŌĆö Retrieval Layer (7-view dense + BM25 + RRF + cross-encoder reranker + songlines) ŌåÆ Belief Layer (current-state filter) ŌåÆ direct-answer or structured summary.
- **Synthesis** (*anum─üna*) ŌĆö gaß╣ćita queries the preserved tuple index exhaustively. **Zero LLM tokens at recall.** The Retrieval Layer still runs in parallel to populate retrieval context, but the synthesis answer is independent of its top-K.

Verified by `tests/belief/test_innovations_compose.py::test_synthesis_intent_independent_of_phase1`, which forces the Retrieval Layer to return `[]` and the gaß╣ćita layer still recovers the canonical $185.

## What's in the wheel

- The PyPI distribution name is **`patha-memory`**. Python import name stays `patha`.
- Wheel metadata: `Name: patha-memory`, `Version: 0.10.3`, `Requires-Python: >=3.11`.
- Three regex extractor false-positive filters (range / hypothetical / negated-purchase) ŌĆö `tests/belief/test_ganita.py::TestFalsePositiveFilters` covers them.
- Hybrid karaß╣ća extractor (`patha.belief.karana.HybridKaranaExtractor`) ŌĆö regex enumerates every `$X`, LLM only labels semantically. Recall preserved; LLM cost paid once at ingest, never at recall.
