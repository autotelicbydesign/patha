# v0.10.2 — first PyPI release

The first public PyPI publish of the v0.10 line. Engineering content is identical to the v0.10.1 git tag (`3a44572`) plus the README/CHANGELOG framing rewrite that demoted v0.10.1 to git-only and led with the architectural distinction instead of benchmark tables.

## Architectural distinction

`Memory.recall()` routes by question intent:

- **Retrieval** (*pratyakṣa*) — Phase 1 (7-view dense + BM25 + RRF + cross-encoder reranker + songlines) → Phase 2 (current-state belief filter) → direct-answer or structured summary.
- **Synthesis** (*anumāna*) — gaṇita queries the preserved tuple index exhaustively. **Zero LLM tokens at recall.** Phase 1 still runs in parallel to populate retrieval context, but the synthesis answer is independent of Phase 1's top-K.

Verified by `tests/belief/test_innovations_compose.py::test_synthesis_intent_independent_of_phase1`, which forces Phase 1 to return `[]` and the gaṇita layer still recovers the canonical $185.

## What's in the wheel

- The PyPI distribution name is **`patha-memory`**. Python import name stays `patha`.
- Wheel metadata: `Name: patha-memory`, `Version: 0.10.2`, `Requires-Python: >=3.11`.
- Three regex extractor false-positive filters (range / hypothetical / negated-purchase) — `tests/belief/test_ganita.py::TestFalsePositiveFilters` covers them.
- Hybrid karaṇa extractor (`patha.belief.karana.HybridKaranaExtractor`) — regex enumerates every `$X`, LLM only labels semantically. Recall preserved; LLM cost paid once at ingest, never at recall.
- Filesystem-native ingest — `patha import obsidian-vault <path>`.

## Pre-flight verification

`scripts/verify_install.sh` creates a throwaway Python 3.11 venv, installs a wheel-or-pip-spec, runs a representative smoke test, and exits non-zero on failure. Used before every PyPI upload in the v0.10 sequence.
