# v0.10.8 — Claude conversation import + CLI fixes (REPL, --version, PATHA_DETECTOR env var)

Real user-feedback-driven release. Stefi tested v0.10.7 on her own machine and surfaced four issues, three of which were genuine bugs:

1. *"Do I have to type `patha ingest` every single freaking time?"* — yes, with the v0.10.7 CLI. **Fixed: new `patha shell` REPL.**
2. *"I set `export PATHA_DETECTOR=full-stack-v8` and the detector didn't change."* — the env var was silently ignored. **Fixed: argparse default now reads `PATHA_DETECTOR`; bad values warn instead of crashing.**
3. *"Patha can't automatically remember all my Claude conversations?"* — correctly observed that v0.10.7's "your memory follows you across tools" line oversold passive observation. Patha is a memory store, not a transcript-scraper. **Fixed two ways: a new `patha import claude-export` retroactively imports past Claude chats, and the README has an explicit honesty paragraph about what "feeds every MCP-compatible tool" actually means.**
4. *"Why are there beliefs in my store I don't recognize?"* — diagnosis-only: this came from accumulated dev/test data in `~/.patha/beliefs.jsonl` from earlier eval runs that wrote to the default store path. The fix is hygiene during testing (use `Memory(path=...)` for tests; never the default in shared dev work). No code change in v0.10.8 for this; documented as a known UX gap.

## What's new

### `patha import claude-export <zip>`

Ingest your Claude conversation history into a Patha store.

Anthropic lets you download all your conversations as a `.zip` from `claude.ai` → Settings → Privacy → Export data. v0.10.8 reads that export and ingests:

- **User messages only.** Assistant replies are skipped — those are Claude's outputs, not yours.
- **Sentence-level by default.** Each user message is split into sentence-ish propositions; each becomes a Patha belief with the original timestamp. Pass `--whole-messages` to ingest each message as one belief instead.
- **Filtered.** Messages shorter than 30 chars, questions to Claude, and obvious commands ("explain X", "rewrite Y") are skipped. Code blocks are stripped before splitting.
- **Capped per message.** Default 10 propositions per user message — prevents one brain-dump from flooding the store.

CLI:
```
patha import claude-export ~/Downloads/data-export.zip
patha import claude-export ~/Downloads/data-export.zip --whole-messages
patha import claude-export ~/Downloads/data-export.zip --verbose
```

Once imported, every existing Patha capability works on the historical beliefs: contradiction detection, supersession, plasticity, history walking, gaṇita synthesis, MCP tool calls. Your past Claude chats become first-class Patha memory, indistinguishable from beliefs you ingest going forward.

### `patha shell` — interactive REPL

Drop the `patha ingest "..."` boilerplate. Type sentences naturally. Prefix `?` (or end with `?`) to ask:

```
$ patha shell
Patha shell. Type a sentence to remember, prefix `?` to ask.
patha> I am vegetarian
  + [added]

patha> ? what do I eat?
  [strategy: direct_answer] [tokens: 0]
  → vegetarian
```

Real usage note (yes, you've heard this before): even the REPL is for scripts, debugging, and one-off interaction. The intended day-to-day interface is the MCP integration — your AI assistant calls `patha_ingest` and `patha_query` for you when you talk to it in Claude Desktop / Cursor / Zed. You should almost never need to type Patha commands directly.

### `PATHA_DETECTOR` env var (and the `install-mcp` follow-on)

```
$ export PATHA_DETECTOR=full-stack-v7
$ patha verify | grep "Selected detector"
Selected detector: full-stack-v7
```

The CLI's `--detector` argument now reads `PATHA_DETECTOR` as its default. Previously the env var was silently ignored — `--detector` defaulted hard-coded to `stub` regardless of env. Bad env-var values now print a clear warning and fall back to `stub` instead of crashing argparse with an opaque choice error.

**Same fix extends to `patha install-mcp`.** Previously, even after running `export PATHA_DETECTOR=full-stack-v8` *and* passing `--detector full-stack-v8` as a global flag, `patha install-mcp` would still write `"PATHA_DETECTOR": "stub"` into the Claude Desktop config. Users had to know to use the (less-discoverable) `--install-detector full-stack-v8` flag, or hand-edit the generated JSON. Now:

```
$ PATHA_DETECTOR=full-stack-v8 patha install-mcp --dry-run | grep PATHA_DETECTOR
      "PATHA_DETECTOR": "full-stack-v8"

$ patha --detector full-stack-v8 install-mcp --dry-run | grep PATHA_DETECTOR
      "PATHA_DETECTOR": "full-stack-v8"

$ patha install-mcp --install-detector full-stack-v8 --dry-run | grep PATHA_DETECTOR
      "PATHA_DETECTOR": "full-stack-v8"
```

The explicit `--install-detector` flag still wins if both are set.

## Documentation honesty pass

The README v0.10.7 said *"The same store feeds every MCP-compatible AI tool: Claude Desktop, Claude Code, Cursor, Zed, Goose. Switch tools mid-project; your memory follows."*

That sentence is technically true (the store is portable) but it implies passive observation that Patha doesn't do. v0.10.8 follows it with a precise clarification: Patha is a memory store, not an observer; facts enter via CLI/REPL/file import/explicit `patha_ingest`. To bulk-import historical Claude chats, use `patha import claude-export`.

## What didn't change

No architectural changes to the Retrieval Layer, Belief Layer, or Articulation Bridge in this release. No detector changes. No benchmark numbers change. This is purely UX, ingest, and documentation.
