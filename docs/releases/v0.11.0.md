# v0.11.0 — narrative synthesis (itihāsa): the third question class, with receipts

The milestone release of the Phase-4 arc. `Memory.recall()` now routes **three** epistemically distinct question classes, each with its own primitive — and the third one ships with the first benchmark of its category, a published held-out generalization gap, and a hardened detector stack whose fixes came from that benchmark's own reveal.

## The headline

- **Retrieval** (*pratyakṣa*) — *"what did I say about X?"* → top-K.
- **Synthesis** (*anumāna*) — *"how much on X total?"* → gaṇita exhaustive arithmetic, zero LLM tokens at recall.
- **Narrative** (*itihāsa*) — *"how has my thinking on X evolved?"* → a temporally-ordered, supersession-tagged walk of the songline graph. New `Recall.narrative` (beats + deterministic through-line), `strategy="narrative"`.

**This release is where the songline graph becomes load-bearing.** Ablations always showed it contributed ≈0 to top-K retrieval — because top-K never needed traversal. Narrative questions are the first class where traversal is the only correct primitive. The philosophy earned its keep when the question class demanded it.

## EvolutionEval — defining the category

No external benchmark measures whether a memory system can reconstruct how thinking evolved (verified: LongMemEval's temporal-reasoning stratum is timestamp arithmetic; knowledge-update is single-fact replacement). EvolutionEval is the first:

- 36 templated dev scenarios + **16 hand-written sealed held-out scenarios in disjoint domains**, across four families grounded in observed real-data failures: `progressive_revelation`, `multi_factor_change`, `perspective_shift`, `reversed_belief_chain`.
- Frozen rubric v1 (routed / coverage / precision / ordering / origin / supersession); deterministic run-to-run (verified byte-identical); artifacts persisted for re-scoring.
- Structural overfit guards: walker frozen before scenario authoring; rubric frozen before the first reported run; 70/30 split committed before any tuning; knob sweeps dev-only.

**The held-out reveal** (run once, numbers frozen as-run): **the temporal core generalizes with ZERO gap** — routing, ordering, and origin identification are 1.000 on all 52 questions, dev *and* held-out, both reported configs. On the shipped default config the dev/held-out delta is statistical noise (coverage −0.007, precision +0.002). Publishing the held-out gap is, to our knowledge, not something any system in this space does.

The full experimental chain — threshold curve with a located cliff, the first reinterpretation-detection measurement (stub 0.000 → v8 0.827), the composed config, the reveal, and the fix validation — is in `docs/benchmarks.md`.

## full-stack-v9 (recommended detector; v7/v8 frozen)

The held-out reveal decomposed its one real gap (v8 supersession 0.808 → 0.625) into three failure classes. All three are fixed in a new stack — v7/v8 stay frozen so published numbers remain reproducible:

1. **Stemming bug** — `_canonicalize_entity` no longer strips `-is/-us/-ss` false plurals ("tennis" → "tenni" is gone).
2. **`SymmetricContradictionDetector`** — belief contradiction is symmetric; NLI is premise/hypothesis-asymmetric (measured pairs: CONTRADICTS 0.97 one way, NEUTRAL 0.99 the other). Reverse-direction adoptions require confidence ≥ 0.90 **and topic overlap ≥ 0.35** — a confidence bar alone cannot filter a model that is *confidently* wrong on unrelated pairs (measured: 0.99 on "squeaky hinge" vs an unrelated reflection). *Virodha* presupposes a shared *viṣaya*: contradiction requires a common locus.
3. **`RevisionPatternDetector`** — three marker families with no prior coverage: resumption ("back on X", "X again now"), settlement ("landed on", "on my own terms"), arrangement ("is/are now", "we do X now"). Additive architecture with embedding topic gates and additive-marker vetoes.

Validation: EvolutionEval dev supersession 0.808 → **0.885** with coverage *and* precision above v8 (strict domination); BeliefEval 300-scenario **347/347** (published v0.10 baseline: 336/347); false-contradiction FP rate **identical to v8** (zero new false positives); quiet on real data (0 revision-pattern fires across the writeups corpus). Generalization claims await held-out **batch 2** — the spent batch cannot be reused as evidence.

## Topic channel

`indexing/topics.py` populates the songline graph's long-vaporware `topic_cluster` field: deterministic agglomerative clustering (cosine, average linkage, first-occurrence label canonicalization) over the **already-computed** v1 "pada" embeddings — no second embedding pass. The walker's on-theme gate accepts cluster-mates, closing the paraphrase gap for abstract themes (which spaCy NER never puts in the entity channel). `PATHA_TOPIC_THRESHOLD` defaults to **0.35**, set by the dev sweep: coverage and precision improve monotonically down to 0.35, and the first metric crack (origin 1.000 → 0.972) appears at 0.25.

## Importer fixes (from real-data dogfooding)

- Frontmatter (`---\ndate: …\n---`) parsed in **all** import modes — plain folder/file imports were ingesting the literal frontmatter block as a belief and stamping everything with file mtimes.
- Per-file sessions for root-level folder imports — a flat 18-file folder previously collapsed into one session, turning the session channel into a near-clique (11,832 edges → 3,928 after the fix).

## Known behaviors, stated plainly

- Chunk-level NLI supersession on long-form essay imports is noisy — this applies to the **whole NLI stack including v9's symmetric path** (instrumented real-data audit: of the 2 symmetric reverse adoptions on the essay corpus, 1 was false, 1 defensible; all trace to NLI being sentence-scale while imported chunks run ~2,000 chars). Supersession semantics are cleanest on atomic facts — the primary MCP usage pattern, where all benchmark validation holds. Holistic fix (propositionize or length-gate before NLI, for the whole stack) is queued for v0.12.
- EvolutionEval rubric v1 scores supersession *recall* only; rubric v2 adds a supersession-*precision* scorer so unexpected edges between gold beliefs can never hide.
- The `multi_factor_change` causal beat (off-theme by substring) and nonmonotonic return-links remain the hardest cases (rb supersession 0.750 on dev under v9-gated).
